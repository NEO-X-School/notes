Peranti [uv](https://docs.astral.sh/uv/) merupakan peranti yang digunakan untuk mengelola instalasi Python, envs, serta paket. Dokumen ini menampilkan petunjuk praktis, bukan petunjuk lengkap. Petunjuk ini bisa digunakan secukupnya untuk membuat direktori (sering disebut dengan workspace) yang spesifik untuk suatu proyek tertentu dengan versi Python tertentu dan paket Python khusus untuk workspace tersebut.

## Instalasi

Petunjuk lengkap instalasi ada di https://docs.astral.sh/uv/getting-started/installation/. Sesuaikan dengan sistem operasi yang digunakan. Berikut adalah contoh di Linux:

```bash
$ curl -LsSf https://astral.sh/uv/install.sh | sh
downloading uv 0.11.2 x86_64-unknown-linux-gnu
installing to /home/bpdp/.local/bin
  uv
  uvx
everything's installed!
$
```

Setelah mengatur env variables untuk uv dan melakukan proses sourcing seperti petunjuk di atas (source $HOME …….. dst), uv  bisa dijalankan sebagai berikut:

```bash
$ uv

Usage: uv [OPTIONS] <COMMAND>

Commands:

  tree     Display the project's dependency tree

$
```

## Update

Update diperlukan jika menginginkan tetap menggunakan versi terbaru dari uv. Berikut adalah caranya:

```bash
$ uv self update
info: Checking for updates...
success: You're already on version v0.11.6 of uv (the latest version).
$
```

Jika terdapat versi baru:

```bash
$ uv self update
info: Checking for updates...
success: Upgraded uv from v0.11.3 to v0.11.6! https://github.com/astral-sh/uv/releases/tag/0.11.6
```


## Membuat *Environment*

Berikut langkah untuk membuat environment:

```bash
~/workspace-01> uv venv
Using CPython 3.14.3 interpreter at: /home/bpdp/.local/bin/python3.14
Creating virtual environment at: .venv
Activate with: source .venv/bin/activate
~/workspace-01> which python 
/usr/bin/python
~/workspace-01>
```

Dari perintah `which`, kita tahu bahwa Python yang kita gunakan masih merupakan bawaan package manager dari sistem operasi, dengan executable berada di `/usr/bin/python`.
Aktifkan dengan perintah `source .venv/bin/acticvate` setelah itu lihat Python yang digunakan.

```bash
~/workspace-01> source .venv/bin/activate
(workspace-01) ~/workspace-01> which python
/path/to/workspace-01/.venv/bin/python
~/workspace-01> 
```

Perhatikan perbedaan letaknya.

**Perhatikan**: pada bagian depan terdapat `(workspace-01)`. Tulisan tersebut menunjukkan workspace yang sedang aktif. Jika ingin keluar dari env, gunakan perintah `deactivate`

```bash
(workspace-01) ~/workspace-01> python
Python 3.14.3 (main, Mar 24 2026, 22:50:36) [Clang 22.1.1 ] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>> <Ctrl-D>
(workspace-01) ~/workspace-01> deactivate 
~/workspace-01> python
ython 3.14.3 (main, Feb 15 2026, 00:49:34) [GCC 15.2.1 20260209] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>> 
~/workspace-01> 
``` 

## Mengelola Paket

Untuk mengelola paket, gunakan `uv pip`. Perintah ini persis sama dengan `pip` di Python seperti biasanya tetapi dikhususkan untuk *workspace* tersebut.

```bash
(workspace-01) ~/workspace-01> uv pip install pandas
Resolved 4 packages in 10ms
Installed 4 packages in 45ms
 + numpy==2.4.4
 + pandas==3.0.2
 + python-dateutil==2.9.0.post0
 + six==1.17.0
(workspace-01) ~/workspace-01>
```

Setelah selesai instalasi:

```bash
(workspace-01) ~/workspace-01> uv pip list
Package         Version
--------------- -----------
numpy           2.4.4
pandas          3.0.2
python-dateutil 2.9.0.post0
six             1.17.0
(workspace-01) ~/workspace-01> uv pip freeze > requirements.txt
(workspace-01) ~/workspace-01> cat requirements.txt 
numpy==2.4.4
pandas==3.0.2
python-dateutil==2.9.0.post0
six==1.17.0
(workspace-01) ~/workspace-01>
```

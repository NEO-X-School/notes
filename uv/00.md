Peranti [uv](https://docs.astral.sh/uv/) merupakan peranti yang digunakan untuk mengelola instalasi Python, envs, serta paket. Dokumen ini menampilkan petunjuk praktis, bukan petunjuk lengkap. Petunjuk ini bisa digunakan secukupnya untuk membuat direktori (sering disebut dengan workspace) yang spesifik untuk suatu proyek tertentu dengan versi Python tertentu dan paket Python khusus untuk workspace tersebut.

## Instalasi

Petunjuk lengkap instalasi ada di https://docs.astral.sh/uv/getting-started/installation/. Sesuaikan dengan sistem operasi yang digunakan. Berikut adalah contoh di Linux:

```bash
$ curl -LsSf https://astral.sh/uv/install.sh | sh
downloading uv 0.11.21 x86_64-unknown-linux-gnu
installing to /home/bpdp/.local/bin
  uv
  uvx
everything's installed!
$
```

## Update

Update diperlukan jika menginginkan tetap menggunakan versi terbaru dari uv. Berikut adalah caranya:

```bash
$ uv self update
info: Checking for updates...
success: You're already on version v0.11.21 of uv (the latest version).
$
```

Jika terdapat versi baru:

```bash
$ uv self update
info: Checking for updates...
success: Upgraded uv from v0.11.20 to v0.11.21! https://github.com/astral-sh/uv/releases/tag/0.11.21
```

**Catatan**: untuk perintah `uv python pin`, bisa menggunakan kolom sebelah kiri (misal: cpython-3.14.3-linux-x86_64-gnu) atau kolom sebelah kanan (misal: /home/bpdp/.local/bin/python3.14). Jika akan menggunakan kolom sebelah kiri dan ada lebih dari satu (lihat pada contoh di atas, cpython-3.14.3-linux-x86_64-gnu ada lebih dari satu), executable akan diambil dari yang paling atas. Jadi, memang lebih aman menggunakan kolom sebelah kanan. Jika menggunakan kolom sebelah kiri dan executable di sebelah kanan belum ada, maka akan dilakukan proses download.

Pada direktori tersebut, akan dibuat file `.python-version`:

```bash

~/workspace-01$ cat .python-version

~/workspace-01$
```

## Membuat *Environment*

Berikut langkah untuk membuat environment:

```bash
~/workspace-01$ uv venv
Using CPython 3.14.6 interpreter at: /home/bpdp/.local/bin/python3.14
Creating virtual environment at: .venv
Activate with: source .venv/bin/activate
~/workspace-01$ which python 
/usr/bin/python
~/workspace-01$
```

Dari perintah `which`, kita tahu bahwa Python yang kita gunakan masih merupakan bawaan package manager dari sistem operasi, dengan executable berada di `/usr/bin/python`. Aktifkan dengan perintah `source .venv/bin/acticvate` setelah itu lihat Python yang digunakan.

```bash
~/workspace-01$ source .venv/bin/activate
(workspace-01) ~/workspace-01$
(workspace-01) ~/workspace-01$ which python
/home/bpdp/tmp/workspace-01/.venv/bin/python
(workspace-01) ~/workspace-01$
```

Perhatikan perbedaan letaknya.

**Perhatikan**: pada bagian depan terdapat `(workspace-01)`. Tulisan tersebut menunjukkan workspace yang sedang aktif. Jika ingin keluar dari env, gunakan perintah `deactivate`

```bash
(workspace-01) ~/workspace-01$ python
Python 3.14.6 (main, Jun 11 2026, 04:03:53) [Clang 22.1.3 ] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>> 
(workspace-01) ~/workspace-01$ deactivate 
(workspace-01) ~/workspace-01$ python
Python 3.14.5 (main, May 14 2026, 13:26:36) [GCC 16.1.1 20260430] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>> 
(workspace-01) ~/workspace-01$
``` 

## Mengelola Paket

Untuk mengelola paket, gunakan `uv pip`. Perintah ini persis sama dengan `pip` di Python seperti biasanya tetapi dikhususkan untuk *workspace* tersebut.

```bash
(workspace-01) ~/workspace-01$ uv pip install pandas
Resolved 4 packages in 216ms
Installed 4 packages in 122ms
 + numpy==2.4.6
 + pandas==3.0.3
 + python-dateutil==2.9.0.post0
 + six==1.17.0
(workspace-01) ~/workspace-01$
```

Setelah selesai instalasi:

```bash
(workspace-01) ~/workspace-01$ uv pip list
Package         Version
--------------- -----------
numpy           2.4.6
pandas          3.0.3
python-dateutil 2.9.0.post0
six             1.17.0
(workspace-01) ~/workspace-01$ uv pip freeze > requirements.txt
(workspace-01) ~/workspace-01$ cat requirements.txt 
numpy==2.4.6
pandas==3.0.3
python-dateutil==2.9.0.post0
six==1.17.0
(workspace-01) ~/workspace-01$
```
